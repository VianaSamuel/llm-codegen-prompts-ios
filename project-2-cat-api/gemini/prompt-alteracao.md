## Prompt Utilizado
```txt
Modifique um aplicativo iOS existente em Swift que consome a API The Cat API, alterando a arquitetura do projeto de MVVM para VIPER. O prompt deve instruir o modelo a: 1) Manter a funcionalidade e os dados da API inalterados. 2) Reestruturar o projeto criando os módulos VIPER (View, Interactor, Presenter, Entity, Router) para a tela principal (listagem de gatos). 3) A View (ViewController) deve ser "burra" e apenas exibir o que o Presenter lhe instrui, e notificar as ações do usuário ao Presenter. 4) O Presenter deve conter a lógica de apresentação e formatar os dados que recebe do Interactor. 5) O Interactor deve ser responsável pela lógica de negócio, como buscar dados da API. 6) A Entity (Model) deve representar a estrutura de dados (ex: CatImage). 7) O Router (ou Wireframe) deve gerenciar a navegação entre telas. 8) Garanta que o código seja legível e siga as boas práticas da arquitetura VIPER, com a interconexão clara entre os módulos via protocolos. 9) O prompt deve instruir o modelo a gerar todo o código modificado em um único bloco, com comentários claros explicando as mudanças e a nova arquitetura, tornando o código auto-suficiente e pronto para ser compilado e executado em um projeto Xcode.
